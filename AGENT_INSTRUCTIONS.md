# Website-Monitor Project - Agent Instructions

These instructions apply to ALL AI agents working on this project.

## 📌 Chat Notes Policy

### REQUIREMENT
Every AI agent MUST create/update session notes when working on this project.

### Location
```
docs/chats_notes/
```

### Naming
```
YYYY-MM-DD_<descriptive_name>.md
```

### Examples
- `2026-02-17_case_inquiry_and_charge_investigation.md`
- `2026-02-18_api_refactoring.md`

### When to Create/Update
- ✅ Starting new work session
- ✅ Major discoveries made
- ✅ New scripts/functions created
- ✅ Session ending or context limit approaching

### What Goes In
See `docs/chats_notes/POLICY.md` for detailed requirements.

Minimum sections:
1. Header (title, date, status)
2. Session Summary (objectives, discoveries)
3. Scripts Created (if any)
4. Technical Insights
5. Notes for Next Session

After creating notes, **UPDATE these files:**

## 💡 Quick Tips

1. Before starting work, check `docs/chats_notes/` for relevant history
2. Link to session notes in INDEX.md for cross-reference
3. Include code examples and usage patterns
4. Note any architectural decisions for future agents
5. Keep notes **factual and actionable**

Session notes are **required infrastructure** for this project.  
They enable smooth transitions between sessions and prevent repeated investigation.

**Non-compliance wastes time and money.**
